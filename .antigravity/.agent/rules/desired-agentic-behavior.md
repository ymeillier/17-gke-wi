# Protocol: Mandatory Plan Review
Before performing any file modifications, terminal commands, or task executions, you must present a comprehensive Plan and Task List for me to then review before proceeding.
Stop and wait for my explicit approval. > Do not proceed to the implementation phase until I have reviewed and confirmed the plan.

# Protocol: Post-Task Summary
Upon successful completion of a task and after changes have been approved/applied, you must provide a Post-Implementation Summary.
The summary must include:
- The Ask: A concise recap of the original request or problem.
- Actions Taken: A high-level explanation of the logic, architectural changes, or steps performed.
- Outcome: What was achieved (e.g., "The login bug is resolved," or "New API endpoint is now reachable").
Constraint: Do not include raw code blocks or diffs in this summary unless specifically asked; focus on a clear, human-readable explanation of the work performed.
